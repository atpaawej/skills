Complete rewrite of `coding-standards` skill with new philosophy and structure.

**What changed:**
- **Philosophy shift**: from passive form-filler to opinionated architect — brings curated best practices, classifies them (MUST/SHOULD/OPTIONAL), recommends before asking
- **Invocation**: from user-invoked to model-invoked — agent fires autonomously when a project needs standards
- **New 6-step flow**: ASSESS → RESEARCH (principle catalog) → PRESENT BASELINE → GRILL (relentless, one-at-a-time) → DRAFT → VERIFY (self-check)
- **Dropped**: `codebase-scan.md` reference, Maintain entry point, ecosystem-conventions, agent-fit-matrix
- **New reference files**: `principles.md` (7-domain principle catalog), `grilling-rhythm.md` (adapted for recommend-first flow)
- **New template**: layered — non-negotiable decisions → rule sections with ✅/❌ examples → boundary system (Always/Ask/Never) → verification self-check
- **Timeless constraint**: document contains rules only — no file paths, no architecture maps, nothing that drifts
- **Leading words**: OCP, Deep Module, Fail Fast, CQS, YAGNI, POLA anchor the skill's behavior
